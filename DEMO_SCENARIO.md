## Возможные вопросы и ответы

### "Покажите самый сложный запрос"

**Открыть app.py, показать query7:**
```python
popular = db.session.query(
    Book.title,
    func.count(Loan.id).label('loan_count')
).join(Loan).group_by(Book.id, Book.title).order_by(func.count(Loan.id).desc()).limit(10).all()
```

**Объяснить:**
- JOIN между Book и Loan
- GROUP BY для группировки по книгам
- COUNT для подсчёта выдач
- ORDER BY DESC для сортировки
- "Есть комментарий - медленно без индекса"

### "Как защищены от SQL injection?"

**Показать код:**
```python
# Плохо:
query = f"SELECT * FROM books WHERE title = '{user_input}'"

# Хорошо (используем):
Book.query.filter(Book.title == user_input).all()
```

"SQLAlchemy автоматически экранирует параметры"

### "Почему Flask, а не Django?"

- Flask легковесный
- Нужен только API + простой UI
- Не нужна админка Django
- Больше контроля над архитектурой

### "Как масштабируется?"

**Показать в docker-compose.yml:**
```yaml
services:
  app:
    deploy:
      replicas: 5  # Можно добавить реплики
```

- Горизонтальное масштабирование через реплики
- Nginx делает load balancing
- PostgreSQL можно вынести на отдельный сервер
- Добавить Redis для кеширования

### "Что бы улучшили?"

1. Добавить тесты (pytest)

3. Celery для асинхронных задач
4. Redis для кеширования
